## Project and research question

The planned system studies whether locally executing cop and thief policies can learn useful pursuit-evasion behavior under partial observability, and whether a scoped CTDE/value-factorization training setup is more stable or effective than Independent Q-Learning (IQL) and non-learning baselines.

The game is formally a competitive POSG because the two roles have separate objectives. The required Dec-POMDP tuple is retained to describe joint state/action/observation mechanics and the training information boundary, but cooperative VDN/QMIX theory will not be presented as a complete solution to the adversarial game.

## Analysis template

### Baseline behavior

The committed smoke comparison uses the same heuristic opponent for every policy within a role. Against the heuristic thief, random/heuristic/learned cops won 1/2/2 of six games. Against the heuristic cop, random/heuristic/learned thieves won 0/4/0 of six games. The learned cop therefore tied rather than exceeded the heuristic, and the learned thief did not show useful behavior. With one evaluation seed set and only six games per condition, these results are pipeline diagnostics only.

### IQL and non-stationarity

Implemented as two replay-backed tabular learners. Each exported agent encodes only its radius-limited observation, masks blocked movement actions, and selects independently. The 40-episode-per-stage sample is a pipeline smoke run only: it is too small, uses no disjoint evaluation seeds, and cannot establish convergence. Simultaneous learning makes each agent's transition dynamics non-stationary; replay can therefore contain data from stale opponent policies.

### CTDE comparison

`IQLTrainer` is CTDE-inspired only at the data/orchestration boundary: `CentralizedTrainingTrace` receives explicit global states, joint actions, both rewards, and terminal flags during training, while action selection and serialized policies receive `LocalObservation` only. No centralized critic consumes that trace yet. VDN and QMIX are not implemented because their standard shared-team-value/IGM assumptions do not directly match this competitive POSG. The extension contract and acceptance gates are documented in [QMIX_EXTENSION.md](QMIX_EXTENSION.md).

### Partial observability and optional memory

Pending. If recurrent models are implemented, report a controlled feed-forward versus GRU/LSTM ablation. Otherwise state that memory remains future work.

### Limitations and threats to validity

Expected threats include few seeds, small grids, opponent overfitting, reward-shaping bias, cooperative-factorization mismatch, hyperparameter selection on evaluation data, and system/network failures. The final report must add observed failures and avoid causal claims unsupported by the experiment design.
